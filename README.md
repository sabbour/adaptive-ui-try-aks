# Ship It

[![CI](https://github.com/sabbour/adaptive-ui-try-aks/actions/workflows/ci.yml/badge.svg)](https://github.com/sabbour/adaptive-ui-try-aks/actions/workflows/ci.yml)

An AI-guided experience for shipping production-ready apps to **AKS Automatic** — the scalable platform for cloud-native and AI-native workloads, built with the [Adaptive UI Framework](https://github.com/sabbour/adaptive-ui-framework).

## What It Does

Ship It is a conversational AI guide that takes you from idea to production deployment on AKS Automatic. No Kubernetes experience required. It offers two tracks:

### Web App or API
Ship web frontends, REST APIs, and microservices (Next.js, Flask, ASP.NET, Go, etc.):
1. **Discover** — Gathers app details: framework, database needs, scaling requirements
2. **Scaffold** — Generates Dockerfile, infrastructure-as-code, routing, and security config
3. **Validate** — Auto-checks and fixes compliance with production best practices
4. **Deploy** — Creates a CI/CD pipeline, commits to your repo, and ships it

### AI Agent
Deploy AI agents with tool-calling, RAG, and self-hosted models:
1. **Discover** — Agent framework, purpose, RAG needs, model selection
2. **Scaffold** — Generates agent container, AI service config, infrastructure
3. **Wire** — Connects to Azure OpenAI or self-hosted models via KAITO, plus databases and search
4. **Deploy** — CI/CD pipeline with secure identity federation

## Key Features

- **Production-ready by default** — Security hardening, health checks, and best-practice infrastructure out of the box
- **Developer-friendly** — No Kubernetes expertise needed; the AI guide translates your intent into deployment config
- **Cost-efficient** — System nodes are free, auto-provisioning picks the cheapest VMs, per-vCPU surcharges keep costs predictable
- **AI-native** — Self-host open-source models (Phi-4, Llama, DeepSeek) via KAITO, or connect to Azure OpenAI. In-cluster RAG pipelines included
- **Deployment Safeguards** — Built-in validator checks manifests against AKS policies (resource limits, probes, security context, etc.)
- **LLM-driven diagrams** — Architecture diagrams generated by the LLM from artifact context (Bicep resources + K8s manifests), with a regenerate button in the viewer
- **Cost estimation** — Inline cost breakdown component showing monthly estimates based on generated artifacts
- **Monaco editor** — VS Code-like file editing with syntax highlighting and web workers
- **Folder tree sidebar** — Organized file hierarchy with Fluent icons, resizable panels

## Layout

Three-panel layout with Azure portal design language:

- **Left panel** — Session sidebar with collapsible folder tree showing generated files (resizable, fills available space)
- **Center panel** — Conversational chat with the deployment agent
- **Right panel** — Monaco editor for viewing/editing Dockerfiles, K8s manifests, Bicep, and architecture diagrams (with regenerate button)

## Running Locally

From the parent workspace root:

```bash
# Start with API backend (recommended)
bash start-app.sh try-aks
```

This starts both the Azure Functions API backend (port 7071) and Vite dev server. The API backend handles CORS proxying for Azure pricing, GitHub OAuth, and Azure AD auth.

Alternatively, start manually:

```bash
# Terminal 1: API backend
cd ../../api && npm run dev

# Terminal 2: Demo app
npm run dev
```

Click the gear icon to connect your OpenAI-compatible LLM endpoint.

For local pack development, symlink local checkouts:

```bash
npm run link:packs
```
